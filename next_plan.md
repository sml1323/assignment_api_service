## 아이디어: 여행 포토북 협업 서비스

### 컨셉
여행 다녀온 후, 함께 간 사람들이 같이 만드는 포토북.
주최자가 사진을 배치하고, 참여자들이 각 페이지에 추억을 남기는 형태.

### 핵심 플로우
1. **주최자**: 여행 사진을 페이지별로 업로드 + 간단한 캡션
2. **공유 링크** 전송 → 참여자들 접속
3. **참여자**: 각 페이지에 텍스트 추가 ("이때 어땠지" 형태의 회상)
   - 페이지당 고정 슬롯 (예: 4개) → 선점 방식
   - 폰트 2-3개 중 선택
   - 글자수 제한 (슬롯 크기에 맞게)
4. **주최자**: 완성된 포토북 미리보기 → 확정 → 인쇄 주문

### 시연용 최소 구현 범위 (3일)
- 주최자: 사진 업로드 + 페이지 구성
- 참여자: 페이지별 텍스트 슬롯에 메시지 작성
- 미리보기 (react-pageflip 재활용)
- Books API + Orders API 연동
- 더미 데이터: 샘플 여행 사진 + 메시지

### 시연 영상에 안 나와도 되는 것 (구현 안 해도 됨)
- 손글씨/그리기 기능 (Canvas)
- 실시간 WebSocket 선점
- 모바일 최적화
- 다중 여행 관리

### 기존 축하책에서 재활용 가능한 것
- FastAPI 백엔드 구조 (models, routes, services)
- React 프론트엔드 구조 (pages, components, api.ts)
- Sweetbook API 연동 (sweetbook.py)
- 이미지 업로드/리사이즈 (image.py)
- 책 미리보기 (BookPreview.tsx + react-pageflip)
- 인증 (admin_token)
- SQLite + WAL

### 변경이 필요한 것

- Contribution 모델 → Page(주최자 사진) + Comment(참여자 메시지)
- 이벤트 생성 → 여행 생성 + 사진 업로드 UI
- 기여자 폼 → 페이지별 텍스트 슬롯 UI
